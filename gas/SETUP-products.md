# んだばいカフェ 商品登録システム — セットアップ手順（秀夫さん用）

妙子先生がGoogleフォームに入力するだけで、ショップ（`shop/index.html`）に商品が自動表示される仕組みです。HTMLを触る人は不要。

```
[妙子先生のスマホ] → [Googleフォーム] → 写真は[Google Drive] / データは[Sheets「商品一覧」]
   → [GAS doGet API] → [shop/index.html が fetch して商品カードを自動生成] → [お客様 → BASEで購入]
```

---

## 初回セットアップ（1回だけ）

### 1. Drive フォルダ
- Googleドライブに「**ndabai-商品写真**」フォルダを作成 → URLの `folders/【ここがID】` をメモ。

### 2. Googleフォーム「んだばい 商品登録フォーム」
項目：
| 項目 | タイプ | 必須 |
|------|--------|------|
| 商品名 | 短文 | ✅ |
| カテゴリ | プルダウン（コーヒー/調味料/ウコン/乾物/椎茸/その他） | ✅ |
| 価格（税込） | 数値 | ✅ |
| 商品説明 | 長文 | ✅ |
| 内容量 | 短文 | ✅ |
| 商品写真 | ファイルアップロード（画像・複数可） | ✅ |
| BASE商品URL | 短文 | ❌ |
| おすすめタグ | チェックボックス（人気/新商品/限定/季節/受賞） | ❌ |
| 備考（ストーリー） | 長文 | ❌ |

- フォーム設定 → 回答 → 「スプレッドシートにリンク」で**新規スプレッドシートを作成**（シート名を「**商品一覧**」にリネーム）。
- ファイルアップロード先が手順1のフォルダになっているか確認。
- 「回答のコピーを送信」をONにすると妙子先生にも控えが届きます。

### 3. スプレッドシート「商品一覧」に列を追加
フォーム連携で A〜J 列が自動生成されます。続けて**手動で見出しを追加**：
- **K列**：写真公開URL（GASが自動記入）
- **L列**：表示フラグ（TRUE/FALSE。非表示にしたい時 FALSE）
- **M列**：並び順（数値・小さい順に表示）

> 列の並び（重要）：A=タイムスタンプ / B=商品名 / C=カテゴリ / D=価格 / E=説明 / F=内容量 / G=写真 / H=BASE URL / I=タグ / J=備考 / K=写真公開URL / L=表示 / M=並び順
> ※ フォーム項目の順序がこの通りになるよう、フォームの質問順を上表に合わせてください。

### 4. GAS（product-api.gs）
- スプレッドシートの「拡張機能 → Apps Script」を開く。
- `product-api.gs` の内容を貼り付け、先頭の **CONFIG** を設定：
  - `SHEET_ID` … スプレッドシートID
  - `PHOTO_FOLDER_ID` … 手順1のフォルダID
  - `ADMIN_EMAIL` … 秀夫さんのメール
- 関数 **`setupTrigger`** を1回実行 → フォーム送信トリガーを設置（権限承認）。

### 5. ウェブアプリとしてデプロイ
- 「デプロイ → 新しいデプロイ → ウェブアプリ」
  - 実行ユーザー：自分／アクセス：**全員**
- 発行URLをコピー → `shared/product-loader.js` の **`PRODUCT_API_URL`** に貼り付け。

### 6. 動作確認
- フォームからテスト送信 → 「商品一覧」に行が追加され、K列に写真URLが入る → ショップページに商品カードが出る。

---

## 運用（毎回）

- **商品追加**：妙子先生がフォームに入力して送信するだけ。即サイト反映。
- **非表示**：シートの **L列を FALSE** に。
- **並び替え**：**M列の数値**を変更（小さいほど上）。
- **BASE URL** 未設定の商品は「準備中」ボタンになります。

### 妙子先生への説明（そのまま送ってOK）
```
このリンクを開いて、商品名・値段・説明・写真を入れて送信するだけ。
サイトに自動で載ります。
面倒なら、写真と情報をメッセンジャーで送ってくれれば私が代わりに入れます。
```

---

## 注意
- Driveの写真共有は「リンクを知っている全員が閲覧可」に自動設定されます（GASが実施）。
- 5MB超の写真はサムネイル（幅800px）配信で軽量化済み。元画像が極端に大きい場合はフォーム前に縮小推奨。
- 反映はリアルタイム（APIが毎回シートを読む）。
